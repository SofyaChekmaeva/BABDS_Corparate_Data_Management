## Техническое видение MVP (RAG-система для поддержки)

### 1. Технологии

| Компонент | Технология | Версия | Назначение |
|-----------|------------|--------|------------|
| **Язык** | Python | 3.10+ | Основной язык разработки |
| **Управление зависимостями** | uv | - | Быстрая установка пакетов |
| **Веб-интерфейс** | Streamlit | 1.28+ | Демо-UI для взаимодействия с системой |
| **Векторизация текста** | scikit-learn (TfidfVectorizer) | 1.3+ | Преобразование чанков в TF-IDF векторы |
| **Матричные операции** | scipy.sparse | - | Хранение и работа с разреженной матрицей TF-IDF |
| **Чанкинг** | Наивный (по символам/параграфам) | - | Разбивка длинных текстов на фрагменты |
| **Поиск** | Косинусное сходство | - | Ранжирование чанков по релевантности запросу |
| **Генерация** | Demo-режим (без LLM) | - | Формирование ответа из найденных чанков |
| **Тестирование** | pytest | 7.4+ | Автоматические тесты |
| **Обработка данных** | pandas | 2.0+ | Работа с CSV-датасетом |

**Почему TF-IDF, а не эмбеддинги?**
- Целью семинара было понять основы поиска информации, поэтому подключать внешние API или импортировать предобученные модели в данном проекте не будем;
- TF-IDF работает без интернета и дополнительных затрат;
- Для данных с четкими ключевыми словами (образения в поддержку к таковым относятся) TF-IDF дает хороший бейзлайн.
